The NAND gate follows the following De Morgan's law: A · B = A + B

So, you can express the same meaning with different symbology:

  • A ∩ B = A ∪ B
  • NOT(A AND B) = (NOT A) OR (NOT B)
  • A · B = A + B
  • !(A && B) === !A || !B the JS way ;)

For clarity, try to replace A and B with true and false and console.log() it. In whichever way you set A and B, the output will always be true.

The NAND gate is the only component you need to build a computer. I am speaking about the computer itself without peripheral devices, so only CPU + RAM
Why?
Because it has the amazing property of functional completeness. It means that based on the way you combine the NAND gates, you can express all possible truth tables (logical NAND, logical conjunction, logical disjunction, logical equality, exlusive disjunction, logical NOR).

But how can I build a gate? What makes a gate a gate?

A gate is made of transistors. Transistors are semiconductor devices that can act as switches or amplifiers.
You can build a circuit on a breadboard to simulate the behaviour of a NAND gate inside a microprocessor. Because they can be combined together on an integrated circuit, it is now possible to put billions of them in one package
